# Video Analysis Reference Templates

Use these templates to guide your analysis based on the video type. Identify the type from the transcript content and apply the relevant template(s). Videos may blend multiple types.

---

## Software Tutorial / Demo

Focus on:
- **Exact menu paths**: File > Settings > Editor > Font Size
- **Keyboard shortcuts** demonstrated or mentioned
- **UI elements**: button locations, panel names, dialog options
- **Configuration values**: settings, flags, options shown on screen
- **Step-by-step procedures**: ordered list of actions to reproduce
- **Version/tool info**: software versions, OS, dependencies mentioned

Frame priorities: Screenshots of UI, settings panels, dialog boxes, terminal output

---

## Coding Tutorial / Live Coding

Focus on:
- **Transcribe code** shown on screen as fenced code blocks with language tags
- **File structure**: project layout, file names, directory organization
- **Dependencies**: packages installed, imports used
- **Terminal commands** run during the tutorial
- **Error messages** shown and how they were resolved
- **Architecture decisions**: why certain patterns or libraries were chosen
- **Git operations**: commits, branches, deployment steps

Frame priorities: Code editor screenshots, terminal output, file trees, browser dev tools

---

## Conference Talk / Presentation

Focus on:
- **Slide content**: key text, bullet points, and diagrams from slides
- **Data/statistics**: numbers, percentages, benchmarks, comparisons
- **Quotes**: notable quotes from the speaker
- **References**: papers, tools, links, books mentioned
- **Argument structure**: thesis, supporting points, conclusion
- **Q&A highlights**: if the video includes audience questions

Frame priorities: Slide content, charts/graphs, diagrams, architecture drawings

---

## Product Review / Unboxing

Focus on:
- **Specifications**: technical specs, dimensions, pricing
- **Comparisons**: how it stacks up against alternatives
- **Pros and cons**: explicit or implied
- **Deal-breakers**: significant issues or limitations
- **Purchase info**: where to buy, pricing tiers, availability
- **Real-world performance**: benchmarks, usage scenarios tested

Frame priorities: Product close-ups, spec sheets, comparison charts, benchmark results

---

## Educational / Explainer

Focus on:
- **Core concepts**: define and explain key terms
- **Analogies**: metaphors or real-world comparisons used
- **Cause and effect**: relationships between concepts
- **Common misconceptions**: mistakes the video addresses
- **Prerequisites**: what you need to know beforehand
- **Further resources**: books, courses, links recommended

Frame priorities: Diagrams, animations, visual explanations, formulas

---

## General / Mixed Format

When the video doesn't fit a specific category:
- Lead with the **core message** or purpose of the video
- Organize by **topics covered** rather than chronologically
- Highlight **actionable information** (things the viewer can do)
- Note any **resources or links** mentioned
- Capture **key quotes** that encapsulate main points
